# CLAUDE.md

このファイルは、Claude Code (claude.ai/code) がこのリポジトリで作業する際のガイドラインです。

## プロジェクト概要

テスト・学習用リポジトリ。Gitチートシートなどの参考資料を含む。

## リポジトリ構成

- `memo/` - 参考資料（Gitチートシートなど）
- `README.md` - プロジェクト説明

### 構成ルール
- 新しいアプリを作成する際は、リポジトリ直下に専用フォルダを作成し、その中にソースコードや関連ファイルをまとめる
  - 例: `todo-app/`, `calculator/` など

## コーディングルール

### HTML
- HTML5 (`<!DOCTYPE html>`) を使用する
- インデントはスペース2つ
- タグ名・属性名は小文字で記述する
- 属性値はダブルクォートで囲む
- セマンティックなHTML要素（`<header>`, `<main>`, `<section>`, `<article>`, `<footer>` など）を適切に使用する
- `lang` 属性を `<html>` タグに指定する

### CSS
- インデントはスペース2つ
- クラス名はケバブケース（`my-class-name`）を使用する
- IDセレクタによるスタイリングは避け、クラスセレクタを使用する
- プロパティはアルファベット順に記述する
- 色の値は小文字の16進数（`#fff`, `#333333`）を使用する
- マジックナンバーを避け、CSS変数（`var(--spacing-md)`）を活用する

### JavaScript
- インデントはスペース2つ
- 文字列はシングルクォートを使用する
- 文末にセミコロンを付ける
- 変数宣言は `const` を優先し、再代入が必要な場合のみ `let` を使用する（`var` は使用しない）
- 命名規則: 変数・関数はキャメルケース（`myVariable`）、クラスはパスカルケース（`MyClass`）、定数は大文字スネークケース（`MAX_COUNT`）
- 比較は厳密等価演算子（`===`, `!==`）を使用する
- アロー関数を優先的に使用する

### Java
- インデントはスペース4つ
- 命名規則: 変数・メソッドはキャメルケース（`myMethod`）、クラスはパスカルケース（`MyClass`）、定数は大文字スネークケース（`MAX_COUNT`）、パッケージは小文字（`com.example.myapp`）
- アクセス修飾子は必ず明示する
- 1クラス1ファイルとし、ファイル名はクラス名と一致させる
- `@Override` アノテーションを必ず付与する

### 共通
- ファイルの末尾には改行を入れる
- 末尾の空白は削除する
- コメントは日本語で記述する

## 対話ルール

- Claudeとの対話はすべて日本語で行う
- コミットメッセージ、ドキュメントも日本語で記述する
